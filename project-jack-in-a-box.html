Jack-in-a-Box Simulator

Year2025
RoleProgrammer
ToolsPython · Lagrangian dynamics

Project Overview

Model a 2D system with at least two rigid bodies, rotational inertia, collisions, and external inputs. Derive equations using Euler–Lagrange mechanics, enforce impact constraints, and visualize the simulation with an animated trajectory.

Code breakdown (PDF)

Approach

For this project, the system was modeled as two solid bodies with their configurations described by their orientation in the plane. The box and the jack have coordinate frames attached, and their positions are related to the world frame through homogeneous transformation matrices:

  1. g_wb — transformation from world to box frame
  2. g_bj — transformation from box to jack frame
  3. g_jr1 – g_jr4 — transformations to the tips of the jack from center
  4. g_bp1 – g_bp4 — transformations to the tips of the sides from box center

I used impact constraints to prevent the jack from passing through the walls of the box. This is done by checking if any of the jack corners overlap the box walls, which are all represented in the world frame through transformation matrices. I also created a constraint equation measuring the distance between the wall of the box and the corner of the jack (φ). When φ(q) = 0 (q being the configuration variable), a collision has occurred. When there is an impact, the jack's velocity component perpendicular to the wall must be consistent with an instantaneous collision.

Both the jack and box are subjected to gravity. These forces are added to the force vector in the Euler–Lagrange equation.

At each timestep in the simulation, the code checks if the constraint has been violated. When a collision is detected, the normal component of the post-impact velocity is set according to the impulse computed via the constraint Jacobian. The velocities of both the jack and the box are updated instantaneously according to the conservation of momentum.
